# Security Policy

## Supported Versions

This repository is a template/starter and does not provide versioned security patch streams.

## Reporting a Vulnerability

1. Do not open a public issue for sensitive vulnerabilities.
2. Report privately to the repository maintainers.
3. Include:
   - Affected file(s) and path(s)
   - Reproduction steps
   - Potential impact
   - Suggested mitigation if available

## Response Expectations

1. Maintainers should acknowledge within 3 business days.
2. Triage and remediation plan should be shared after validation.
3. Public disclosure should happen only after a fix is available.
