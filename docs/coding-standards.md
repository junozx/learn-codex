# Coding Standards

## Python

1. Use `ruff format` and `ruff check`.
2. Prefer explicit type hints on public functions.
3. Keep files and functions small and focused.

## React + Semi

1. Build UI with Semi components first, then custom styles.
2. Keep components focused by feature/domain.
3. Ensure responsive behavior for desktop and mobile.

## Testing

1. Add tests for behavior changes.
2. Keep unit tests deterministic.
3. Add regression tests for bug fixes.

## Pull Requests

1. Reference spec folder (`specs/NNN-name`).
2. Include acceptance evidence.
3. Keep scope minimal and focused.
